# Contributing Guidelines

Thank you for your interest in contributing to the Offline-First Music Identification App.

This project is **founder-owned** and **vision-locked**. Contributions are welcome only if they align with the Product Definition Document (PDD).

---

## Governance

* **Founder / Product Owner:** Feroze Linx
* The Founder retains final decision authority on scope, features, and direction.
* All contributions are subject to review for alignment with the PDD and Core Principles.

---

## Core Principles (Non‑Negotiable)

Contributions must respect the following:

* Offline‑first operation is mandatory
* The app adapts to the environment, not the user
* No assumption of constant connectivity
* Failure is informative and improves the system
* User trust, consent, and privacy come first

Any proposal that contradicts these principles will not be accepted.

---

## What You Can Contribute (At This Stage)

* Documentation improvements
* Architecture proposals
* Audio processing research notes
* Offline‑first design patterns
* Privacy‑preserving approaches

Code contributions will be accepted only after an MVP architecture is finalized.

---

## Data & Privacy Rules

* **No real user audio** may be added to this repository
* **No datasets** may be introduced without explicit Founder approval
* All experiments must be clearly labeled and consent‑safe

---

## Submitting Contributions

1. Fork the repository
2. Create a feature or documentation branch
3. Reference the relevant PDD section in your proposal
4. Submit a pull request with a clear explanation

---

## Intellectual Property

By contributing, you acknowledge:

* The original concept and product vision belong to the Founder
* Contributions may be used, modified, or excluded at the Founder’s discretion
* No ownership transfer is implied by contribution

---

## Questions

For clarification or proposals, open an issue tagged **discussion**.

Thank you for respecting the vision and helping build something meaningful.
